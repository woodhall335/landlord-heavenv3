# Blog Topic Consolidation Plan (Phase 7)

## 1) Tenant rent non-payment / arrears cluster
**Decision:** Differentiate harder + canonical intent hierarchy.

- Keep core pages separate by intent stage:
  - “tenant stopped paying rent” = triage + next steps
  - “evict tenant not paying rent” = possession workflow
  - “money claim unpaid rent” = debt recovery workflow
- Keep supporting blog posts as children; route them to one of the above as primary CTA.
- If two child posts have near-identical H1/intent, canonicalize the weaker page to the stronger parent.

## 2) Section 21 cluster
**Decision:** Keep separate + canonicalize overlap.

- Anchor pillar: `/section-21-notice-guide`
- Child pages should be split into:
  - validity/compliance checklists
  - service mechanics
  - post-notice court process
- Any duplicate “what is section 21” explainers should be merged or canonicalized to strongest URL.

## 3) Section 8 cluster
**Decision:** Keep separate + structure by grounds/court stage.

- Anchor pillar: `/section-8-notice-guide`
- Child pages should be differentiated by:
  - grounds selection
  - arrears-specific evidence
  - hearing/contested route
- Consolidate near-duplicate “section 8 process” pages into one primary process URL + supporting child pages.

## 4) Repeated UK-wide compliance explainers
**Decision:** Leave as supporting children but tighten differentiation.

- Keep each by compliance object (deposit, EPC, gas, EICR, right-to-rent, licensing).
- Remove generic repetition from intros and route each to eviction dependencies in-body.
- Use canonical tags where two pages compete for same head term with minimal unique value.

## 5) Nation-specific process explainers
**Decision:** Keep separate as jurisdiction hubs.

- Wales, Scotland, and Northern Ireland pages should remain standalone (different law = justified separation).
- Consolidate duplicates within each jurisdiction where page purpose overlaps heavily.

## Consolidation implementation order
1. Canonicalize duplicate metadata targets first.
2. Merge low-unique child pages with overlapping H1 + same user intent.
3. Retain jurisdiction-specific legal pages even with similar structures.
